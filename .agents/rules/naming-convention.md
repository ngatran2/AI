# Naming Convention Rules

> All output files generated by agents MUST follow these naming conventions.

## General Pattern

```
[UC-ID]_[feature-name]_[type]_[date-or-version].[ext]
```

### Components

| Component        | Format              | Example            | Notes                                    |
| :--------------- | :------------------ | :----------------- | :--------------------------------------- |
| `UC-ID`          | `UC-XXX`            | `UC-101`           | Use Case ID from requirement document    |
| `feature-name`   | `lowercase-kebab`   | `user-login`       | Short descriptive name, max 3-4 words    |
| `type`           | See table below     | `extracted`        | Indicates the output type                |
| `date`           | `YYYYMMDD`          | `20260324`         | Date the file was created                |
| `version`        | `vN`                | `v1`               | Version number for finalized files       |

## Output Types

| Type              | Suffix Value          | Extension | Used By            |
| :---------------- | :-------------------- | :-------- | :----------------- |
| Audited Req       | `audited`             | `.md  `   | requirement-reviewer |
| Scenario          | `scenarios`           | `.md`     | qa-engineer        |
| Test Case (HL)    | `testcases-hl`        | `.xlsx`   | qa-engineer        |
| Test Case (DET)   | `testcases-det`       | `.xlsx`   | qa-engineer        |
| Plan              | `plan`                | `.md`     | planner            |
| Research          | `research`            | `.md`     | researcher         |
| Brainstorm        | `brainstorm`          | `.md`     | brainstormer       |
| Execution Result  | `res_[UC-ID]_..._testcases_res` | `.xlsx`   | quality-control    |
| Report            | `[report-type]`       | `.md`     | Any agent          |

> **Test Case Level Guide:**
> - `testcases-hl` (High-Level): Smoke/Sanity — chỉ các Happy Path chính + Critical Error. Dùng cho kiểm tra nhanh hotfix, build mới.
> - `testcases-det` (Detail-Level): Full Regression — bao gồm toàn bộ Edge Cases, BVA, RBAC, Validation chi tiết. Dùng cho kiểm thử toàn diện trước release.

## File Name Examples
# Step 1: Audited requirements
UC-101_user-login_audited_20260403_v1.md

# Step 2: Scenarios
UC-101_user-login_scenarios_20260403_v1.md
UC-101_user-login_scenarios_20260403_v2.md   ← revised version

# Step 3A: Test cases (High-level)
UC-101_user-login_testcases-hl_20260403_v1.xlsx
UC-101_user-login_testcases-hl_20260403_v2.xlsx

# Step 3B: Test cases (Detail-level)
UC-101_user-login_testcases-det_20260403_v1.xlsx
UC-101_user-login_testcases-det_20260403_v2.xlsx

# Planner, Researcher, Brainstormer Outputs
UC-101_user-login_plan_20260324_v1.md
UC-101_user-login_research_20260324_v1.md
UC-101_user-login_brainstorm_20260324_v1.md

# Reports
audit-report_UC-101_20260324.md
brainstorm-report_UC-101_20260324.md

## Version Control
- Files use `v[N]` version suffix — they are **immutable** once created.
- When creating a new version, increment `N` (v1 → v2 → v3).
- Never delete or overwrite a version.
